# Use Cases

## Use Case 1: Draft Incident Report

A user enters incident details such as date, time, location, people involved, and narrative notes. The system organizes the information into a structured draft report.

## Use Case 2: Missing Information Checklist

A user enters partial case details. The system reviews the input and lists missing information, such as location, incident type, witness names, or evidence notes.

## Use Case 3: Case Summary

A user provides long notes from an investigation. The system creates a short summary that can be reviewed and edited by the user.
